## この本が生まれた背景

あるPRレビューの日、私は気づいた。

同じコードを見て、Claude APIは「N+1クエリが発生しています」と1秒で指摘した。私は15分かけてコードを読んで、ようやく同じ問題に気づいた。

一方、別のPRでは逆のことが起きた。Claudeは「このコードは正しく動作します」と回答した。しかし私には引っかかりがあった。*なぜここでこのモデルを使うのか。3ヶ月後、要件が変わったときにこの設計は持つのか。*

これが本書の出発点だ。

**AIは何を見て、何を見ていないのか。人間は何を見て、何を見落としているのか。**

この問いに正面から答える技術書が、日本にはほとんどない。

## この本で得られること

1. **概念の整理**：AIレビューと人間レビューの根本的な違いを言語化できる
2. **実装スキル**：Claude APIを使ったレビューシステムを自分で構築できる
3. **設計知識**：AIと人間の役割分担を設計できる
4. **運用ノウハウ**：チームへの段階的な導入方法を理解できる

## 各章の概要

### 第1章：AIと人間のコードレビュー — 根本的な違い（無料）

「AIに任せれば良い」は間違いであり、「AIは使えない」も間違い。両者が「なぜ」異なるのかを、認知科学と実測データで解説する。

### 第2章：AIが瞬時に検出する5つのコードパターン

バグ・セキュリティ脆弱性・パフォーマンス問題・スタイル違反・重複コード。AIが圧倒的に得意な領域を実例で体感する。

### 第3章：人間レビュアーが重視する5つの視点

アーキテクチャ適合性・保守性・ビジネスロジックの妥当性・チームの慣習・将来の拡張性。人間にしか判断できない領域を整理する。

### 第4章：Claude APIでレビューシステムを実装する

PRのdiffを受け取り、コメントを生成してGitHubに投稿する完全なシステムを実装する。

### 第5章：プロンプト設計 — レビュー品質を決める核心

「バグを探して」では使い物にならない。レビュー観点を構造化したプロンプトの設計方法を、A/Bテストのデータとともに解説する。

### 第6章：設計レビュー — アーキテクチャの問題を早期発見

単一責任原則・依存関係・インターフェース設計をAIでチェックする手法。設計フェーズでのAI活用を詳説する。

### 第7章：セキュリティレビュー — AIで脆弱性を自動検出

SQLインジェクション・XSS・認証不備・パストラバーサル。OWASPリストに基づく自動セキュリティレビューを実装する。

### 第8章：CI/CD統合 — GitHub Actions × Claude API

PRをトリガーにAIレビューが自動実行されるパイプラインを構築する。コスト管理とレート制限の実践的な対処法も含む。

### 第9章：チーム導入ガイド — AIと人間のハイブリッドワークフロー

「AIのレビューが間違っていた」「レビュアーが不要になった」という誤解を防ぐ。段階的な導入プロセスと、チームの心理的安全性の保ち方を解説する。

### 第10章：実測データ公開 — AIレビュー導入でバグ率-60%の実際

6ヶ月間の運用データ、コスト、チームへの影響を包み隠さず公開する。「本当に効果があるのか」への回答。

### 付録A：Claude Reviewプロンプト集

設計レビュー・セキュリティレビュー・パフォーマンスレビュー・可読性レビューのプロンプトテンプレート25本。

### 付録B：AIコードレビュー 導入チェックリスト

チームへの導入前・導入中・定着後の確認項目30点。

## 著者について

Python × AI開発を専門とするエンジニア。`notecreator`（AI記事自動生成システム）を実際に構築・運用し、Claude APIの実践的な活用法を日々探求している。本書で紹介するコードは全て本番環境で検証済みのものです。
