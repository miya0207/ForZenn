## AIレビューの出力例

以下のような脆弱なコードをAIに送ると：

```python
def get_user(username: str):
    query = f"SELECT * FROM users WHERE username = '{username}'"
    return db.execute(query)
```

数秒でこのようなレビューコメントが返ってきます：

```
🔴 重大: SQLインジェクション脆弱性

【問題】
username パラメータが直接クエリ文字列に埋め込まれています。
攻撃者が username に "admin' OR '1'='1" を入力すると、
全ユーザーのデータが漏洩します。

【修正例】
def get_user(username: str):
    return db.execute(
        "SELECT * FROM users WHERE username = ?",
        (username,)
    )

【参考】OWASP A03:2021 - Injection
```

## この本が生まれた背景

あるPRレビューの日、私は気づいた。

同じコードを見て、Claude APIは「N+1クエリが発生しています」と1秒で指摘した。私は15分かけてコードを読んで、ようやく同じ問題に気づいた。

一方、別のPRでは逆のことが起きた。Claudeは「このコードは正しく動作します」と回答した。しかし私には引っかかりがあった。*なぜここでこのモデルを使うのか。3ヶ月後、要件が変わったときにこの設計は持つのか。*

これが本書の出発点だ。

**AIは何を見て、何を見ていないのか。人間は何を見て、何を見落としているのか。**

この問いに正面から答える技術書が、日本にはほとんどない。

## 各章の概要

### 第1章：AIコードレビューを5分で体験する（無料）

まず動かしてみる。脆弱なコードをClaude APIに送り、実際のレビュー結果を見る。

### 第2章：AIコードレビューの仕組み（無料）

AIがコードをどう読んでいるか。パターンマッチングとLLM推論の仕組みを、実測データで解説。

### 第3章：AIが得意なレビュー領域

バグ・セキュリティ・パフォーマンス・スタイル・設計パターン。AIが高精度で検出できる5＋1の領域。

### 第4章：人間が担うべきレビュー

アーキテクチャ・ビジネスルール・保守性。AIが間違える実例と、人間が必ず確認すべき観点。

### 第5章：Claude APIでレビューBotを作る

`review_bot.py`・`github_client.py`・`claude_client.py` を実装。30行の最小実装から本番品質まで。

### 第6章：プロンプトエンジニアリング

「バグを探して」では使えない。Role/Task/Format/Constraintsの4要素でレビュー精度を最大化する。

### 第7章：セキュリティレビュー

SQLi・XSS・パストラバーサル・コマンドインジェクション。OWASP対応の自動セキュリティチェックを実装。

### 第8章：GitHub Actions自動レビュー

PRトリガーで自動実行される完全なCI/CDパイプライン。コスト制御・レート制限対策を含む本番対応YAML。

### 第9章：チーム導入ガイド

3フェーズの段階的導入。「AIが人間を置き換える」誤解への答えと、よくある反発への対応。

### 第10章：ROIの測定

バグ率・レビュー時間・APIコスト。6ヶ月間の実測データで効果を定量化する方法。

### 第11章：失敗事例と対策

AI幻覚・悪いプロンプト・盲信による失敗。現場で起きた問題と、具体的な緩和戦略。

### 付録A：Claude Reviewプロンプト集25本

設計・セキュリティ・パフォーマンス・可読性・テスタビリティの各観点プロンプトとPython実装。

### 付録B：AIコードレビュー導入チェックリスト30項目

導入前・導入中・定着後の確認項目。必須🔴と推奨🟡で優先度を明示。
