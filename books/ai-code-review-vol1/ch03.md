## 3.1 領域①：バグ（論理エラー・型エラー）

## 3.2 領域②：セキュリティ脆弱性

**AIの得意度: ★★★★★**（OWASP Top 10への対応が高精度）

### SQLインジェクション

```python
# ❌ 危険
def get_user(username: str):
    query = f"SELECT * FROM users WHERE username = '{username}'"
    return db.execute(query)
# 攻撃: username = "admin' OR '1'='1"

# ✅ 安全（プレースホルダ）
def get_user(username: str):
    return db.execute(
        "SELECT * FROM users WHERE username = ?", (username,)
    )
```

### XSS（Cross-Site Scripting）

```python
# ❌ 危険（Flaskで変数を直接HTMLに埋め込む）
@app.route("/search")
def search():
    q = request.args.get("q", "")
    return f"<h1>検索: {q}</h1>"  # <script>alert(1)</script> が実行される

# ✅ 安全（Jinjaテンプレートの自動エスケープ）
@app.route("/search")
def search():
    q = request.args.get("q", "")
    return render_template_string("<h1>検索: {{ q }}</h1>", q=q)
```

### パストラバーサル

```python
# ❌ 危険
def read_file(filename: str) -> str:
    return (Path("/var/www/uploads") / filename).read_text()
# filename="../../etc/passwd" でシステムファイルが読まれる

# ✅ 安全
def read_file(filename: str) -> str:
    base = Path("/var/www/uploads").resolve()
    target = (base / filename).resolve()
    if not str(target).startswith(str(base)):
        raise ValueError("不正なパス")
    return target.read_text()
```

### コマンドインジェクション

```python
# ❌ 危険
import os
def convert(filename: str):
    os.system(f"convert {filename} out.png")
# filename="in.jpg; rm -rf /"

# ✅ 安全（リスト形式）
import subprocess
def convert(filename: str):
    subprocess.run(["convert", filename, "out.png"], check=True)
```

## 3.3 領域③：パフォーマンス問題

**AIの得意度: ★★★★☆**

```python
# ❌ N+1クエリ（最多検出パターン）
def get_dashboard(user_ids: list[int]) -> list[dict]:
    users = db.query(User).filter(User.id.in_(user_ids)).all()
    result = []
    for user in users:
        posts = db.query(Post).filter(Post.user_id == user.id).all()  # N+1
        result.append({"user": user.name, "posts": len(posts)})
    return result

# ✅ JOINで1クエリに
from sqlalchemy import func
def get_dashboard(user_ids: list[int]) -> list[dict]:
    rows = (
        db.query(User, func.count(Post.id).label("cnt"))
        .outerjoin(Post, Post.user_id == User.id)
        .filter(User.id.in_(user_ids))
        .group_by(User.id)
        .all()
    )
    return [{"user": u.name, "posts": cnt} for u, cnt in rows]


# ❌ ループ内の重複計算
def process_logs(lines: list[str]) -> list[str]:

    config = load_config()  # 1回だけ
```

**AIの得意度: ★★★☆☆**（diff内で完結する問題のみ）

```
  ✅ God Class（1クラスにメソッドが20個以上）
```

### ASTで循環依存を検出

```python
# circular_dep_detector.py
import ast
from pathlib import Path

def get_imports(filepath: Path, package: str) -> list[str]:
    try:
        tree = ast.parse(filepath.read_text(encoding="utf-8"))
    except (SyntaxError, UnicodeDecodeError):
        return []
    result = []
    for node in ast.walk(tree):
        mod = None
        if isinstance(node, ast.Import):
            mod = node.names[0].name if node.names else None
        elif isinstance(node, ast.ImportFrom) and node.module:
            mod = node.module
        if mod and mod.startswith(package):
            result.append(mod)
    return result

def detect_cycles(root: str, package: str) -> list[list[str]]:
    project = Path(root)
    graph: dict[str, list[str]] = {
        str(f.relative_to(project)).replace("/", ".").removesuffix(".py"):
        get_imports(f, package)
        for f in project.rglob("*.py")
    }
    cycles, visited, stack = [], set(), set()

    def dfs(node: str, path: list[str]):
        visited.add(node); stack.add(node)
        for nb in graph.get(node, []):
            if nb not in visited:
                dfs(nb, path + [nb])
            elif nb in stack and nb in path:
                cycles.append(path[path.index(nb):] + [nb])
        stack.discard(node)

    for m in graph:
        if m not in visited:
            dfs(m, [m])
    return cycles
```

## 実務Tips

**Tip 1: モデルを使い分けてコストを最適化**
- バグ・スタイル・重複 → `claude-haiku-4-5`（速度優先、コスト約1/10）
- セキュリティ・設計 → `claude-sonnet-4-6`（精度優先）

**Tip 2: 大きなファイルは関数単位に分割**
1,000行超のファイルは `ast.parse` で関数・クラス単位に分割して送ると精度が向上する。

**Tip 3: false positiveを蓄積・改善**
誤指摘をjsonlに記録し、月次でプロンプトに「以下は除外してください」として追記する。

## まとめ

AIが得意な6つの領域：

1. **バグ** — ミュータブルデフォルト引数・浮動小数点比較など構造から判定できる問題
2. **セキュリティ** — SQLi・XSS・パストラバーサル・コマンドインジェクション
3. **パフォーマンス** — N+1クエリ・ループ内重複計算
4. **可読性** — マジックナンバー・複雑な否定条件（Lintを補完）
5. **重複** — 構造的類似性の認識
6. **設計パターン** — diff内で完結するSOLID違反・循環依存
