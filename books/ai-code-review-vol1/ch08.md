# 第8章: CI/CD統合 — GitHub Actions × Claude API

:::message
**この章で学べること**
- GitHub ActionsでAIレビューを完全自動化するワークフローの実装
- PRのdiffをGitHub APIから取得してClaudeに送る一連の流れ
- コスト・レート制限・大規模PRへの対策を含む本番レベルの設計
:::

## 8.1 全体アーキテクチャ

本章では、PRが作成・更新されるたびにAIレビューが自動実行される仕組みを構築する。

```
[PR作成/更新]
     ↓
[GitHub Actions トリガー]
     ↓
[GitHub API で diff 取得]
     ↓
[チャンク分割（大規模PR対応）]
     ↓
[Claude API でレビュー実行]
     ↓
[レート制限・リトライ制御]
     ↓
[PR にコメント投稿]
```

最終的なワークフローは以下の機能を持つ：

- PRのdiff自動取得
- 1PRあたりの最大トークン数制限（コスト制御）
- 指数バックオフによるレート制限対策
- 大規模PRのチャンク処理
- Secretsによる安全なAPI Key管理

## 8.2 GitHub Actions Workflow 完全版

`.github/workflows/ai-review.yml` の完全実装を示す。

```yaml
# .github/workflows/ai-review.yml
name: AI Code Review

on:
  pull_request:
    types: [opened, synchronize, reopened]
    # ドキュメントのみの変更はスキップ
    paths-ignore:
      - "**.md"
      - "docs/**"
      - ".github/**.md"

# 同一PRの古いジョブをキャンセルして最新のみ実行
concurrency:
  group: ai-review-${{ github.event.pull_request.number }}
  cancel-in-progress: true

permissions:
  contents: read
  pull-requests: write  # PRコメント投稿に必要

jobs:
  ai-review:
    name: Claude AI Review
    runs-on: ubuntu-latest
    timeout-minutes: 10  # 無限ループ防止

    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          # PRのdiffを取得するために必要
          fetch-depth: 0

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"
          cache: "pip"

      - name: Install dependencies
        run: |
          pip install anthropic PyGithub tenacity

      - name: Run AI Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          PR_NUMBER: ${{ github.event.pull_request.number }}
          REPO_FULL_NAME: ${{ github.repository }}
          # コスト制限設定（環境変数で制御可能にする）
          MAX_TOKENS_PER_PR: "8000"
          MAX_FILES_PER_PR: "20"
        run: |
          python .github/scripts/ai_review.py

      - name: Upload review log
        if: always()  # 失敗時もログを保存
        uses: actions/upload-artifact@v4
        with:
          name: ai-review-log-${{ github.event.pull_request.number }}
          path: /tmp/ai_review_*.json
          retention-days: 30
```

:::note
**`GITHUB_TOKEN` は自動提供される**
`secrets.GITHUB_TOKEN` はGitHub Actionsが自動的に提供するトークンです。別途Secretsを設定する必要はありません。`ANTHROPIC_API_KEY` のみ手動でSecretsに追加してください。
:::

## 8.3 レビュースクリプトの実装

`.github/scripts/ai_review.py` の完全実装：

```python
"""
GitHub Actions上で動作するAIコードレビュースクリプト。

必要な環境変数:
  ANTHROPIC_API_KEY  - Claude APIキー
  GITHUB_TOKEN       - GitHub API認証トークン
  PR_NUMBER          - レビュー対象のPR番号
  REPO_FULL_NAME     - "owner/repo" 形式のリポジトリ名
  MAX_TOKENS_PER_PR  - 1PRあたりの最大入力トークン数（デフォルト: 8000）
  MAX_FILES_PER_PR   - 1PRあたりの最大ファイル数（デフォルト: 20）
"""

import json
import logging
import os
import sys
import time
from dataclasses import dataclass
from datetime import datetime
from pathlib import Path
from typing import Optional

import anthropic
from github import Github
from tenacity import (
    retry,
    retry_if_exception_type,
    stop_after_attempt,
    wait_exponential,
)

# ログ設定
logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(message)s",
)
logger = logging.getLogger(__name__)

# 設定定数
REVIEW_PROMPT_SYSTEM = """
あなたはシニアソフトウェアエンジニアとして、コードレビューを行います。

## レビューの観点（優先順位順）
1. 🔴 重大: バグ・セキュリティ脆弱性・データ破壊のリスク
2. 🟡 警告: パフォーマンス問題・エラーハンドリング不備・設計上の懸念
3. 🔵 提案: 可読性改善・リファクタリング候補・ベストプラクティス

## 出力フォーマット（厳守）
各指摘は以下の形式で記述してください：

**[優先度] ファイル名:行番号付近 — 問題の要約**

問題の説明（1〜2文）

```言語
// 修正前（問題のある箇所のみ抜粋）
悪いコード
```

```言語
// 修正後
良いコード
```

## 制約
- 指摘は最大10件に絞り、重要なものを優先してください
- コードが良ければ「問題なし」と1行で返してください
- ファイルパスと行番号は diff から読み取ってください
- 日本語で記述してください
""".strip()


@dataclass
class ReviewConfig:
    """レビュー設定."""

    max_tokens_per_pr: int = 8000
    max_files_per_pr: int = 20
    max_retries: int = 3
    chunk_size_tokens: int = 4000  # チャンク処理時の1チャンクあたりのトークン数


@dataclass
class PRDiff:
    """PRのdiff情報."""

    total_additions: int
    total_deletions: int
    files: list[dict]
    raw_diff: str
    is_truncated: bool = False


def get_pr_diff(repo_name: str, pr_number: int, config: ReviewConfig) -> PRDiff:
    """
    GitHub APIからPRのdiffを取得する。

    Args:
        repo_name: "owner/repo" 形式のリポジトリ名
        pr_number: PR番号
        config: レビュー設定

    Returns:
        PRDiff: diffの情報

    Raises:
        ValueError: PR番号が不正な場合
        GithubException: API呼び出し失敗時
    """
    token = os.environ["GITHUB_TOKEN"]
    g = Github(token)
    repo = g.get_repo(repo_name)
    pr = repo.get_pull(pr_number)

    files = list(pr.get_files())
    total_additions = sum(f.additions for f in files)
    total_deletions = sum(f.deletions for f in files)

    logger.info(
        f"PR #{pr_number}: {len(files)} files, "
        f"+{total_additions}/-{total_deletions} lines"
    )

    # ファイル数制限の適用
    is_truncated = False
    if len(files) > config.max_files_per_pr:
        logger.warning(
            f"ファイル数が制限({config.max_files_per_pr})を超えているため、"
            f"最初の {config.max_files_per_pr} ファイルのみ処理します"
        )
        files = files[: config.max_files_per_pr]
        is_truncated = True

    # diffテキストの構築
    diff_parts = []
    current_tokens = 0

    for file in files:
        if file.patch is None:
            # バイナリファイル等はスキップ
            continue

        file_diff = f"## {file.filename}\n```diff\n{file.patch}\n```\n"
        # 簡易トークン推定（実際は文字数/4が目安）
        estimated_tokens = len(file_diff) // 4

        if current_tokens + estimated_tokens > config.max_tokens_per_pr:
            logger.warning(
                f"トークン制限({config.max_tokens_per_pr})に達したため、"
                f"以降のファイルをスキップします"
            )
            is_truncated = True
            break

        diff_parts.append(file_diff)
        current_tokens += estimated_tokens

    raw_diff = "\n".join(diff_parts)

    return PRDiff(
        total_additions=total_additions,
        total_deletions=total_deletions,
        files=[{"filename": f.filename, "status": f.status} for f in files],
        raw_diff=raw_diff,
        is_truncated=is_truncated,
    )


@retry(
    retry=retry_if_exception_type(anthropic.RateLimitError),
    wait=wait_exponential(multiplier=1, min=4, max=60),
    stop=stop_after_attempt(3),
    before_sleep=lambda retry_state: logger.warning(
        f"レート制限に達しました。{retry_state.next_action.sleep:.1f}秒後にリトライ..."
    ),
)
def call_claude_with_retry(
    client: anthropic.Anthropic,
    diff_text: str,
    pr_title: str,
    pr_description: str,
) -> str:
    """
    指数バックオフ付きでClaude APIを呼び出す。

    tenacityライブラリによりRateLimitError時に自動リトライします。
    最大3回リトライ、待機時間は4秒→8秒→16秒（上限60秒）。

    Args:
        client: Anthropic APIクライアント
        diff_text: レビュー対象のdiffテキスト
        pr_title: PRのタイトル
        pr_description: PRの説明文

    Returns:
        レビュー結果テキスト
    """
    user_message = f"""
## PRの概要
**タイトル**: {pr_title}
**説明**: {pr_description or "（説明なし）"}

## コードの変更内容（diff）
{diff_text}

---
上記のコード変更をレビューしてください。
""".strip()

    response = client.messages.create(
        model="claude-haiku-4-5",  # コスト最適化: Haikuを使用
        max_tokens=2048,
        system=REVIEW_PROMPT_SYSTEM,
        messages=[{"role": "user", "content": user_message}],
    )

    return response.content[0].text


def review_in_chunks(
    client: anthropic.Anthropic,
    pr_diff: PRDiff,
    pr_title: str,
    pr_description: str,
    config: ReviewConfig,
) -> str:
    """
    大規模PRをチャンクに分割してレビューする。

    全diffが1回のAPI呼び出しに収まる場合は直接レビュー。
    大きい場合はファイルごとに分割して複数回呼び出し、結果を統合する。

    Args:
        client: Anthropic APIクライアント
        pr_diff: PRのdiff情報
        pr_title: PRのタイトル
        pr_description: PRの説明文
        config: レビュー設定

    Returns:
        統合されたレビュー結果
    """
    # diffのテキスト量を確認
    total_chars = len(pr_diff.raw_diff)
    estimated_tokens = total_chars // 4

    logger.info(f"diff推定トークン数: {estimated_tokens}")

    if estimated_tokens <= config.chunk_size_tokens:
        # 小規模PR: 一括処理
        logger.info("一括レビューを実行します")
        return call_claude_with_retry(
            client, pr_diff.raw_diff, pr_title, pr_description
        )

    # 大規模PR: チャンク処理
    logger.info(f"大規模PR検出: チャンク処理を開始します")
    chunk_results = []
    current_chunk = []
    current_chunk_tokens = 0

    # ファイル単位でチャンク分割
    diff_lines = pr_diff.raw_diff.split("\n## ")
    chunks = []
    current = []
    current_tokens = 0

    for section in diff_lines:
        section_tokens = len(section) // 4
        if current_tokens + section_tokens > config.chunk_size_tokens and current:
            chunks.append("\n## ".join(current))
            current = [section]
            current_tokens = section_tokens
        else:
            current.append(section)
            current_tokens += section_tokens

    if current:
        chunks.append("\n## ".join(current))

    logger.info(f"チャンク数: {len(chunks)}")

    for i, chunk in enumerate(chunks, 1):
        logger.info(f"チャンク {i}/{len(chunks)} をレビュー中...")
        chunk_result = call_claude_with_retry(
            client,
            chunk,
            f"{pr_title} (パート {i}/{len(chunks)})",
            pr_description,
        )
        chunk_results.append(f"### パート {i}/{len(chunks)} のレビュー\n\n{chunk_result}")
        # チャンク間の短い待機（レート制限の予防的措置）
        if i < len(chunks):
            time.sleep(1)

    return "\n\n---\n\n".join(chunk_results)


def post_review_comment(
    repo_name: str,
    pr_number: int,
    review_text: str,
    pr_diff: PRDiff,
) -> None:
    """
    レビュー結果をPRにコメントとして投稿する。

    既存のAIレビューコメントがあれば更新し、なければ新規作成する。
    これにより、同じPRにレビューコメントが溜まることを防ぐ。

    Args:
        repo_name: "owner/repo" 形式のリポジトリ名
        pr_number: PR番号
        review_text: レビュー結果テキスト
        pr_diff: PRのdiff情報（統計情報の表示に使用）
    """
    token = os.environ["GITHUB_TOKEN"]
    g = Github(token)
    repo = g.get_repo(repo_name)
    pr = repo.get_pull(pr_number)

    # コメントヘッダー（既存コメント検索のキーとして使用）
    comment_header = "## 🤖 AIコードレビュー (Claude)"
    truncation_warning = ""
    if pr_diff.is_truncated:
        truncation_warning = (
            "\n\n> ⚠️ **注意**: PRのサイズが大きいため、一部のファイルのみをレビューしています。"
            "残りのファイルは手動でレビューしてください。\n"
        )

    full_comment = f"""{comment_header}

{truncation_warning}
**変更統計**: +{pr_diff.total_additions} / -{pr_diff.total_deletions} 行 | {len(pr_diff.files)} ファイル

---

{review_text}

---
*このレビューはClaude AI (claude-haiku-4-5)が自動生成しました。*
*最終更新: {datetime.now().strftime('%Y-%m-%d %H:%M:%S')} UTC*
"""

    # 既存のAIレビューコメントを探す
    existing_comment = None
    for comment in pr.get_issue_comments():
        if comment.body.startswith(comment_header):
            existing_comment = comment
            break

    if existing_comment:
        logger.info("既存のレビューコメントを更新します")
        existing_comment.edit(full_comment)
    else:
        logger.info("新規レビューコメントを投稿します")
        pr.create_issue_comment(full_comment)


def save_review_log(
    pr_number: int,
    review_result: str,
    pr_diff: PRDiff,
) -> None:
    """レビュー結果をJSONファイルに保存する（アーティファクト用）."""
    log_data = {
        "timestamp": datetime.now().isoformat(),
        "pr_number": pr_number,
        "stats": {
            "additions": pr_diff.total_additions,
            "deletions": pr_diff.total_deletions,
            "files": len(pr_diff.files),
            "is_truncated": pr_diff.is_truncated,
        },
        "review_length": len(review_result),
    }
    log_path = Path(f"/tmp/ai_review_{pr_number}.json")
    log_path.write_text(json.dumps(log_data, ensure_ascii=False, indent=2))
    logger.info(f"レビューログを保存: {log_path}")


def main() -> int:
    """
    メイン処理。

    Returns:
        終了コード (0: 成功, 1: エラー)
    """
    # 環境変数の読み込み
    api_key = os.environ.get("ANTHROPIC_API_KEY")
    repo_name = os.environ.get("REPO_FULL_NAME")
    pr_number_str = os.environ.get("PR_NUMBER")

    if not all([api_key, repo_name, pr_number_str]):
        logger.error("必要な環境変数が設定されていません")
        return 1

    pr_number = int(pr_number_str)

    config = ReviewConfig(
        max_tokens_per_pr=int(os.environ.get("MAX_TOKENS_PER_PR", "8000")),
        max_files_per_pr=int(os.environ.get("MAX_FILES_PER_PR", "20")),
    )

    try:
        # PRのdiffを取得
        logger.info(f"PR #{pr_number} のdiffを取得中...")
        token = os.environ["GITHUB_TOKEN"]
        g = Github(token)
        repo = g.get_repo(repo_name)
        pr = repo.get_pull(pr_number)

        pr_diff = get_pr_diff(repo_name, pr_number, config)

        if not pr_diff.raw_diff.strip():
            logger.info("レビュー対象のdiffがありません（バイナリファイルのみなど）")
            return 0

        # Claudeでレビュー実行
        logger.info("Claude AIによるレビューを開始...")
        client = anthropic.Anthropic(api_key=api_key)

        review_result = review_in_chunks(
            client=client,
            pr_diff=pr_diff,
            pr_title=pr.title,
            pr_description=pr.body or "",
            config=config,
        )

        # PRにコメント投稿
        post_review_comment(repo_name, pr_number, review_result, pr_diff)

        # ログ保存
        save_review_log(pr_number, review_result, pr_diff)

        logger.info("AIレビュー完了")
        return 0

    except anthropic.APIConnectionError as e:
        logger.error(f"Claude API接続エラー: {e}")
        return 1
    except anthropic.AuthenticationError:
        logger.error("ANTHROPIC_API_KEY が無効です")
        return 1
    except Exception as e:
        logger.exception(f"予期しないエラー: {e}")
        return 1


if __name__ == "__main__":
    sys.exit(main())
```

## 8.4 Secretsの安全な管理

### 必須Secretsの設定手順

```bash
# GitHub CLI を使った設定
gh secret set ANTHROPIC_API_KEY --body "sk-ant-..."

# または GitHub Web UI から設定
# Settings → Secrets and variables → Actions → New repository secret
```

:::message alert
**絶対にやってはいけないこと**

```yaml
# ❌ NG: ワークフローにAPIキーを直書き
env:
  ANTHROPIC_API_KEY: "sk-ant-api03-..."

# ❌ NG: ログに出力される可能性がある
- run: echo "API Key is $ANTHROPIC_API_KEY"

# ✅ OK: Secretsから参照
env:
  ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```
:::

### 組織レベルのSecrets管理

複数リポジトリでAPIキーを共有する場合は、組織レベルのSecretsを使う：

```bash
# 組織の全リポジトリに適用
gh secret set ANTHROPIC_API_KEY \
  --org YOUR_ORG \
  --visibility all \
  --body "sk-ant-..."

# 特定リポジトリのみに適用
gh secret set ANTHROPIC_API_KEY \
  --org YOUR_ORG \
  --repos "repo1,repo2" \
  --body "sk-ant-..."
```

## 8.5 コスト制御の実装

### トークン使用量の見積もり

```python
def estimate_review_cost(diff_chars: int) -> dict:
    """
    レビューのおおよそのコストを見積もる。

    Args:
        diff_chars: diffの文字数

    Returns:
        コスト見積もり情報
    """
    # Claude Haiku の価格 (2024年時点)
    INPUT_COST_PER_1M = 0.25   # $0.25 per 1M input tokens
    OUTPUT_COST_PER_1M = 1.25  # $1.25 per 1M output tokens

    estimated_input_tokens = diff_chars // 4
    estimated_output_tokens = 500  # レビュー結果の典型的な長さ

    input_cost = (estimated_input_tokens / 1_000_000) * INPUT_COST_PER_1M
    output_cost = (estimated_output_tokens / 1_000_000) * OUTPUT_COST_PER_1M
    total_cost = input_cost + output_cost

    return {
        "estimated_input_tokens": estimated_input_tokens,
        "estimated_output_tokens": estimated_output_tokens,
        "estimated_cost_usd": round(total_cost, 6),
        "estimated_cost_jpy": round(total_cost * 150, 3),  # 概算
    }


# 使用例
cost = estimate_review_cost(diff_chars=10000)
print(f"推定コスト: ${cost['estimated_cost_usd']} (約¥{cost['estimated_cost_jpy']})")
# → 推定コスト: $0.000688 (約¥0.103)
```

:::note
**月間コストの目安**

| チーム規模 | 日次PR数 | 月次PR数 | 月次コスト(概算) |
|-----------|---------|---------|----------------|
| 個人 | 2件 | 40件 | $0.03 |
| 5人チーム | 15件 | 300件 | $0.25 |
| 20人チーム | 60件 | 1200件 | $1.00 |
| 100人チーム | 300件 | 6000件 | $5.00 |

Claude Haikuを使うと非常に安価です。コストよりもAPIのレート制限の方が先に問題になります。
:::

## 🚨 本番落とし穴

### 落とし穴①：`GITHUB_TOKEN` の権限不足

デフォルトの `GITHUB_TOKEN` は読み取り専用の場合がある。

```yaml
# ✅ 明示的に書き込み権限を付与する
permissions:
  contents: read
  pull-requests: write
```

### 落とし穴②：フォークからのPRでSecretsが使えない

フォークリポジトリからのPRでは、セキュリティ上の理由で `secrets` が空になる。

```yaml
# ✅ 対策: フォークPRを明示的に処理する
jobs:
  ai-review:
    # フォークPRではスキップ（Secretsが利用できないため）
    if: github.event.pull_request.head.repo.full_name == github.repository
```

### 落とし穴③：大量のコメントがPRを埋める

PRを何度も更新するたびに新規コメントが追加されると、PRが読めなくなる。本章の実装では既存コメントを検索して**更新**する方式を採用している。

### 落とし穴④：タイムアウトでジョブが失敗する

Claude APIのレスポンスが遅い場合、デフォルトのタイムアウトで失敗することがある。

```yaml
# ✅ タイムアウトを適切に設定
jobs:
  ai-review:
    timeout-minutes: 10  # 10分で強制終了
```

## 実務Tips

**Tip 1: draft PRではレビューをスキップする**

```yaml
on:
  pull_request:
    types: [opened, synchronize, reopened, ready_for_review]

jobs:
  ai-review:
    if: github.event.pull_request.draft == false
```

**Tip 2: ラベルで動的に制御する**

```yaml
jobs:
  ai-review:
    # "skip-ai-review" ラベルがついたPRはスキップ
    if: "!contains(github.event.pull_request.labels.*.name, 'skip-ai-review')"
```

**Tip 3: レビュー結果をSlackに通知する**

```yaml
- name: Notify Slack on review failure
  if: failure()
  uses: slackapi/slack-github-action@v1
  with:
    payload: |
      {"text": "⚠️ AIレビューが失敗しました: ${{ github.event.pull_request.html_url }}"}
  env:
    SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}
```

## まとめ

本章では、PRトリガーのGitHub Actions WorkflowにClaude APIを統合する完全な実装を示した。

- **完全なYAMLワークフロー**: concurrencyで古いジョブをキャンセル、permissionsで最小権限
- **安全なSecrets管理**: 絶対にAPIキーをコードに書かない
- **コスト制御**: `MAX_TOKENS_PER_PR` でトークン量を制限
- **レート制限対策**: `tenacity` ライブラリによる指数バックオフ付きリトライ
- **大規模PR対応**: ファイル単位のチャンク分割処理
- **コメント重複防止**: 既存コメントを更新する実装
