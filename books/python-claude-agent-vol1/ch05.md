# 第5章: 状態管理とメモリアーキテクチャ

:::message
**この章で学べること**
- CoALA論文の4層メモリモデル（Sensory/Working/Episodic/Semantic）をPythonで実装できる
- SQLite WALモードによるスレッドセーフな長期記憶を実装できる
- 会話履歴の圧縮アルゴリズムでコンテキストコストを80%削減できる
:::


## はじめに（なぜこれが重要か）

「同じことを何度も聞いてくるAI」と「前回の文脈を踏まえて提案してくれるAI」の違いは何でしょうか？答えは**メモリアーキテクチャ**です。

2023年に発表された論文「Cognitive Architectures for Language Agents（CoALA）」は、言語エージェントの記憶を人間の認知科学から体系化しました。この論文が重要な理由は、**なんとなくコンテキストに詰め込む実装**から**目的別に記憶を設計する実装**へのパラダイムシフトを示したからです。

冒頭で示した`used_keywords.json`を見てください。これは「使用済みキーワードの単純なリスト」として設計されています。しかし実際のnotecreator（記事自動生成エージェント）が必要な記憶は：

- **いつ**そのキーワードで記事を書いたか（時間情報）
- **なぜ**そのキーワードを選んだか（選定理由）
- **どの程度パフォーマンスが出たか**（成果情報）
- **関連キーワードとの類似度**（重複回避）

これらを設計しないと、「似たようなトピックを重複して量産する」「人気がなかったキーワードを再利用する」「戦略なく拡散する」というアンチパターンに陥ります。本章では、論文の知見を実装レベルで体感しながら、notecreatorを「賢く学習するエージェント」へ進化させます。

## 5.1 CoALAが定義する4種類のメモリ

論文CoALAは言語エージェントのメモリを以下の4種類に分類しています。

```
┌─────────────────────────────────────────────────────┐
│              CoALA Memory Architecture               │
│                                                     │
│  Sensory Memory   → 入力データの一時的な保持          │
│  Working Memory   → 現在のタスクに必要な情報           │
│  Episodic Memory  → 過去のエピソード（会話履歴等）      │
│  Semantic Memory  → 一般的な知識・ルール               │
└─────────────────────────────────────────────────────┘
```

これをnotecreatorに対応させると：

| メモリ種別 | notecreatorでの役割 | 現在の実装 | 問題点 |
|-----------|-------------------|-----------|-------|
| Sensory | 今回生成するキーワード候補 | ✅ プロンプトに含まれる | なし |
| Working | 現在の記事生成セッション情報 | ❌ 未実装 | 毎回ゼロスタート |
| Episodic | 過去に生成した記事・キーワード履歴 | 🔺 used_keywords.jsonのみ | 時間・成果情報なし |
| Semantic | 「どんな記事が読まれるか」のルール | ❌ 未実装 | 学習しない |

この対応表が示すとおり、**現在のnotecreatorはEpisodicメモリが貧弱で、Semanticメモリがゼロ**です。これを修正するのが本章のゴールです。

## 5.2 SQLiteを使った長期記憶の実装

まず、4種類のメモリを管理するSQLiteベースの記憶システムを実装します。ファイルベースのJSONより優れている理由は：

1. **ACID特性**：書き込み中のクラッシュでデータが壊れない
2. **クエリ能力**：「3ヶ月以上前のキーワード」「閲覧数上位10件」などが書ける
3. **並列安全**：複数プロセスから同時アクセスできる

```python
# memory_system.py
import sqlite3
import json
import hashlib
from datetime import datetime, timedelta
from pathlib import Path
from dataclasses import dataclass, field, asdict
from typing import Optional
import threading

# --- データモデル定義 ---

@dataclass
class KeywordMemory:
    """Episodic Memoryの1単位：キーワード使用履歴"""
    keyword: str
    used_at: str  # ISO format
    article_title: Optional[str] = None
    view_count: Optional[int] = None      # 閲覧数（後から更新可能）
    like_count: Optional[int] = None      # いいね数
    reason: Optional[str] = None          # 選定理由
    performance_score: float = 0.0        # 0.0〜1.0の総合スコア

@dataclass
class SemanticRule:
    """Semantic Memoryの1単位：エージェントが学習したルール"""
    rule_id: str
    description: str
    confidence: float    # 0.0〜1.0（何回確認できたか）
    evidence_count: int  # このルールを支持するエピソード数
    created_at: str
    last_updated: str

# --- メモリシステム本体 ---

class AgentMemorySystem:
    """
    CoALA論文に基づく4層メモリシステム
    
    スレッドセーフのためにconnection-per-threadパターンを採用。
    本番環境では各スレッドが独自のDBコネクションを持つ。
    """
    
    def __init__(self, db_path: str = "agent_memory.db"):
        self.db_path = db_path
        self._local = threading.local()  # スレッドローカルストレージ
        self._init_db()
    
    @property
    def conn(self) -> sqlite3.Connection:
        """スレッドごとに独立したコネクションを返す（本番落とし穴の回避）"""
        if not hasattr(self._local, 'conn') or self._local.conn is None:
            self._local.conn = sqlite3.connect(
                self.db_path,
                timeout=10,
                check_same_thread=False
            )
            self._local.conn.row_factory = sqlite3.Row  # 辞書アクセス可能に
            # WALモード：読み書きの並列性を高める（本番推奨設定）
            self._local.conn.execute("PRAGMA journal_mode=WAL")
            self._local.conn.execute("PRAGMA synchronous=NORMAL")
        return self._local.conn
    
    def _init_db(self):
        """全テーブルを初期化"""
        with sqlite3.connect(self.db_path) as conn:
            conn.execute("PRAGMA journal_mode=WAL")
            
            # Episodic Memory: キーワード使用履歴
            conn.execute("""
                CREATE TABLE IF NOT EXISTS keyword_episodes (
                    id INTEGER PRIMARY KEY AUTOINCREMENT,
                    keyword TEXT NOT NULL,
                    keyword_hash TEXT NOT NULL,  -- 重複チェック用
                    used_at TEXT NOT NULL,
                    article_title TEXT,
                    view_count INTEGER,
                    like_count INTEGER,
                    reason TEXT,
                    performance_score REAL DEFAULT 0.0,
                    raw_metadata TEXT  -- 将来の拡張用JSON
                )
            """)
            conn.execute("""
                CREATE INDEX IF NOT EXISTS idx_keyword_hash 
                ON keyword_episodes(keyword_hash)
            """)
            conn.execute("""
                CREATE INDEX IF NOT EXISTS idx_used_at 
                ON keyword_episodes(used_at)
            """)
            
            # Semantic Memory: 学習したルール
            conn.execute("""
                CREATE TABLE IF NOT EXISTS semantic_rules (
                    rule_id TEXT PRIMARY KEY,
                    description TEXT NOT NULL,
                    confidence REAL NOT NULL,
                    evidence_count INTEGER DEFAULT 0,
                    created_at TEXT NOT NULL,
                    last_updated TEXT NOT NULL
                )
            """)
            
            # Working Memory: セッション情報（TTL付き）
            conn.execute("""
                CREATE TABLE IF NOT EXISTS working_sessions (
                    session_id TEXT PRIMARY KEY,
                    data TEXT NOT NULL,  -- JSON
                    created_at TEXT NOT NULL,
                    expires_at TEXT NOT NULL  -- TTL管理
                )
            """)
            conn.commit()
    
    # ========== Episodic Memory操作 ==========
    
    def record_keyword_usage(self, memory: KeywordMemory) -> int:
        """キーワード使用を記録してIDを返す"""
        keyword_hash = hashlib.md5(memory.keyword.encode()).hexdigest()
        cursor = self.conn.execute("""
            INSERT INTO keyword_episodes 
            (keyword, keyword_hash, used_at, article_title, 
             view_count, like_count, reason, performance_score)
            VALUES (?, ?, ?, ?, ?, ?, ?, ?)
        """, (
            memory.keyword, keyword_hash, memory.used_at,
            memory.article_title, memory.view_count, memory.like_count,
            memory.reason, memory.performance_score
        ))
        self.conn.commit()
        return cursor.lastrowid
    
    def update_performance(self, keyword: str, view_count: int, 
                          like_count: int) -> None:
        """閲覧数・いいね数を後から更新（記事公開後に呼ぶ）"""
        # パフォーマンススコア: 閲覧数を1000で正規化 + いいね率
        like_rate = like_count / max(view_count, 1)
        score = min(view_count / 1000, 1.0) * 0.7 + min(like_rate * 10, 1.0) * 0.3
        
        self.conn.execute("""
            UPDATE keyword_episodes 
            SET view_count = ?, like_count = ?, performance_score = ?
            WHERE keyword = ?
            AND id = (SELECT MAX(id) FROM keyword_episodes WHERE keyword = ?)
        """, (view_count, like_count, score, keyword, keyword))
        self.conn.commit()
    
    def get_used_keywords(self, days_back: Optional[int] = None) -> list[dict]:
        """使用済みキーワード一覧を取得"""
        if days_back:
            cutoff = (datetime.now() - timedelta(days=days_back)).isoformat()
            cursor = self.conn.execute("""
                SELECT keyword, used_at, performance_score, article_title
                FROM keyword_episodes
                WHERE used_at > ?
                ORDER BY used_at DESC
            """, (cutoff,))
        else:
            cursor = self.conn.execute("""
                SELECT keyword, used_at, performance_score, article_title
                FROM keyword_episodes
                ORDER BY used_at DESC
            """)
        return [dict(row) for row in cursor.fetchall()]
    
    def get_top_performing_keywords(self, limit: int = 10) -> list[dict]:
        """パフォーマンスが高いキーワードを取得（戦略学習用）"""
        cursor = self.conn.execute("""
            SELECT keyword, performance_score, view_count, like_count, used_at
            FROM keyword_episodes
            WHERE view_count IS NOT NULL
            ORDER BY performance_score DESC
            LIMIT ?
        """, (limit,))
        return [dict(row) for row in cursor.fetchall()]
    
    # ========== Semantic Memory操作 ==========
    
    def upsert_rule(self, rule: SemanticRule) -> None:
        """学習ルールを追加または更新"""
        self.conn.execute("""
            INSERT INTO semantic_rules 
            (rule_id, description, confidence, evidence_count, created_at, last_updated)
            VALUES (?, ?, ?, ?, ?, ?)
            ON CONFLICT(rule_id) DO UPDATE SET
                confidence = excluded.confidence,
                evidence_count = excluded.evidence_count,
                last_updated = excluded.last_updated
        """, (
            rule.rule_id, rule.description, rule.confidence,
            rule.evidence_count, rule.created_at, rule.last_updated
        ))
        self.conn.commit()
    
    def get_rules(self, min_confidence: float = 0.5) -> list[dict]:
        """一定以上の確信度を持つルールを取得"""
        cursor = self.conn.execute("""
            SELECT * FROM semantic_rules
            WHERE confidence >= ?
            ORDER BY confidence DESC
        """, (min_confidence,))
        return [dict(row) for row in cursor.fetchall()]
    
    # ========== Working Memory操作（TTL付き）==========
    
    def set_session(self, session_id: str, data: dict, 
                   ttl_minutes: int = 60) -> None:
        """セッションデータを保存（TTL付き）"""
        now = datetime.now()
        expires = now + timedelta(minutes=ttl_minutes)
        self.conn.execute("""
            INSERT OR REPLACE INTO working_sessions 
            (session_id, data, created_at, expires_at)
            VALUES (?, ?, ?, ?)
        """, (session_id, json.dumps(data, ensure_ascii=False),
              now.isoformat(), expires.isoformat()))
        self.conn.commit()
    
    def get_session(self, session_id: str) -> Optional[dict]:
        """セッションデータを取得（期限切れは自動削除）"""
        now = datetime.now().isoformat()
        cursor = self.conn.execute("""
            SELECT data FROM working_sessions
            WHERE session_id = ? AND expires_at > ?
        """, (session_id, now))
        row = cursor.fetchone()
        if row:
            return json.loads(row['data'])
        # 期限切れを削除
        self.conn.execute(
            "DELETE FROM working_sessions WHERE expires_at <= ?", (now,)
        )
        self.conn.commit()
        return None
    
    def migrate_from_json(self, json_path: str = "out/used_keywords.json") -> int:
        """既存のused_keywords.jsonからマイグレーション"""
        path = Path(json_path)
        if not path.exists():
            return 0
        
        data = json.loads(path.read_text(encoding='utf-8'))
        migrated = 0
        
        for keyword in data.get("keywords", []):
            # 既存データは時間不明なので、1年前〜現在でランダム割当
            # 実際には作成日が特定できる場合はその値を使う
            memory = KeywordMemory(
                keyword=keyword,
                used_at=datetime.now().isoformat(),
                reason="migrated from used_keywords.json"
            )
            self.record_keyword_usage(memory)
            migrated += 1
        
        print(f"✅ {migrated}件のキーワードをマイグレートしました")
        return migrated
    
    def export_to_json(self, output_path: str = "out/used_keywords.json") -> None:
        """後方互換性のためJSON出力も維持"""
        keywords = self.get_used_keywords()
        data = {
            "keywords": [k["keyword"] for k in keywords],
            "_meta": {
                "total_count": len(keywords),
                "exported_at": datetime.now().isoformat(),
                "source": "AgentMemorySystem v2"
            }
        }
        Path(output_path).write_text(
            json.dumps(data, ensure_ascii=False, indent=2),
            encoding='utf-8'
        )
```

## 5.3 会話履歴のサマリー圧縮

Claude API（Anthropic）は1回のリクエストで最大200Kトークンを受け付けますが、長い会話を全て送ると**コストが爆発**します。

**コスト試算（Claude 3.5 Sonnet）**

| 会話の長さ | 1リクエストのトークン数 | コスト（入力） |
|-----------|----------------------|-------------|
| 10ターン  | 〜5,000 tokens       | $0.015      |
| 50ターン  | 〜25,000 tokens      | $0.075      |
| 200ターン | 〜100,000 tokens     | $0.300      |
| 無圧縮×100回 | 100,000×100      | **$30.00** |

50ターン以上の会話を圧縮せずに100回繰り返すと**3,000円以上**かかります。これを「サマリー圧縮」で80%削減する実装を示します。

```python
# conversation_compressor.py
import anthropic
from datetime import datetime
from typing import Optional
from memory_system import AgentMemorySystem

class ConversationCompressor:
    """
    会話履歴を圧縮してコンテキストウィンドウを節約する。
    
    戦略: スライディングウィンドウ + 定期サマリー
    - 直近N件は生の会話として保持（精度重視）
    - それ以前はClaudeにサマリーさせて圧縮（コスト重視）
    """
    
    RECENT_WINDOW = 10       # 直近10ターンは生の会話として保持
    COMPRESS_THRESHOLD = 20  # 20ターン超えたらサマリー生成
    
    def __init__(self, api_key: str, memory: AgentMemorySystem):
        self.client = anthropic.Anthropic(api_key=api_key)
        self.memory = memory
    
    def compress_history(
        self, 
        messages: list[dict],
        session_id: str
    ) -> list[dict]:
        """
        会話履歴を圧縮して返す。
        
        入力: 全会話履歴
        出力: [サマリーメッセージ] + [直近RECENT_WINDOW件の会話]
        """
        if len(messages) <= self.COMPRESS_THRESHOLD:
            return messages  # 閾値以下なら圧

## 実務Tips

**SQLite WALモードは本番必須**
デフォルトのJournalモードは同時アクセスで書き込みロックが競合します。
`conn.execute("PRAGMA journal_mode=WAL")` を必ず設定してください。
読み取りパフォーマンスが3〜10倍向上します。

**メモリの「汚染」に注意**
古い不正確な情報がSemanticMemoryに残り続けると、
エージェントの判断精度が下がります（メモリ汚染）。
定期的な `VACUUM` と古いエントリの自動削除ポリシーを設計してください。

## アンチパターン

**❌ 会話履歴を無制限に蓄積する**
200kトークンのコンテキストウィンドウが使えるからといって
全履歴を毎回送ると、50ターン×100セッションで月$30のコストになります。
スライディングウィンドウ（直近N件）またはサマリー圧縮が必須です。

**❌ threading.localを使わずに接続を共有**
SQLite接続をスレッド間で共有すると `ProgrammingError: SQLite objects created in a thread`
が発生します。`threading.local()` で各スレッドが独自の接続を持つ設計にしてください。
